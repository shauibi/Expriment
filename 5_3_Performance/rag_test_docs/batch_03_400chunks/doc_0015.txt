微服务架构将单体应用拆分为一组小型、独立部署的服务，每个服务围绕特定的业务能力构建，拥有独立的数据库和技术栈。服务间通过轻量级通信协议（如HTTP/REST、gRPC或消息队列）进行交互。微服务架构的优势包括独立部署和扩展、技术异质性、故障隔离和团队自治。然而，分布式系统的固有复杂性（网络延迟、数据一致性、服务发现、分布式事务）是需要权衡的代价。

计算机视觉是人工智能的核心领域之一，目标是使计算机能够从图像和视频中获取高层次的语义理解。主要任务包括图像分类、目标检测、语义分割、实例分割、姿态估计和图像生成。残差网络（ResNet）通过跳跃连接解决了深层网络的梯度消失问题，使得训练数百层的神经网络成为可能。视觉Transformer（ViT）将Transformer架构应用于图像领域，将图像分割为固定大小的图像块作为输入序列。

Serverless（无服务器计算）是云计算的一种执行模型，云提供商动态管理机器资源的分配。开发者只需关注业务代码逻辑，无需关心服务器的配置、扩展和维护。AWS Lambda、Azure Functions和Google Cloud Functions是主流Serverless平台。Serverless架构对于间歇性工作负载和事件驱动型应用特别有优势，但在冷启动延迟、状态管理和供应商锁定方面也存在挑战。

大语言模型（LLM）是近年来人工智能领域最具突破性的进展之一。GPT-4、Claude、Gemini等模型拥有数千亿参数，在海量文本和代码数据上训练，展现出强大的语言理解、推理和生成能力。涌现能力（Emergent Abilities）是指当模型规模超过某个阈值时突然展现出的新能力，包括少样本学习、思维链推理和代码生成等。然而，大语言模型也面临幻觉问题、事实性错误和偏见等挑战。

持续集成与持续交付（CI/CD）是现代DevOps实践的核心。持续集成要求开发者频繁将代码变更合并到主干分支，每次提交自动触发构建和测试流程，尽早发现集成问题。持续交付确保代码库始终处于可发布状态，任何通过测试的变更都可以一键部署到生产环境。Jenkins、GitHub Actions、GitLab CI和CircleCI是常用的CI/CD工具。蓝绿部署和金丝雀发布是降低部署风险的常用策略。

容器技术是现代云原生架构的基石。Docker通过将应用及其依赖打包到轻量级、可移植的容器中，实现了开发和生产环境的一致性。Kubernetes（K8s）是谷歌开源的容器编排平台，提供了自动部署、弹性伸缩、负载均衡、服务发现和存储编排等丰富的集群管理功能。Helm作为Kubernetes的包管理器，简化了复杂应用的部署和管理。服务网格（Service Mesh）如Istio将微服务间的通信治理下沉到基础设施层。

强化学习是一种通过与环境交互来学习最优策略的机器学习方法。智能体在环境中执行动作，观察状态变化和获得的奖励信号，通过试错学习最大化累积奖励。Q-learning和策略梯度是两类经典算法。深度强化学习将深度神经网络与强化学习相结合，AlphaGo和AlphaZero在围棋等复杂博弈游戏中取得了超越人类顶尖水平的成就。近端策略优化（PPO）和软演员-评论家（SAC）是当前应用最广泛的深度强化学习算法。

边缘计算将计算和数据存储推向网络边缘，靠近数据源和用户终端，以降低延迟、减少带宽消耗并增强数据隐私保护。与集中式的云计算相比，边缘计算更适用于对实时性要求高的场景，如工业互联网、自动驾驶和增强现实。5G通信技术的高带宽和低延迟特性为边缘计算的发展提供了网络基础。云边协同是当前分布式计算的重要研究方向。

自然语言处理（NLP）是人工智能与语言学的交叉学科，致力于让计算机理解、生成和推理人类语言。典型任务包括文本分类、命名实体识别、关系抽取、情感分析、机器翻译、问答系统和文本摘要。预训练语言模型如BERT、GPT系列通过在海量无标注文本上进行自监督预训练，再在下游任务上微调，极大提升了各项NLP任务的性能。提示学习（Prompt Learning）作为一种新的范式，通过设计合适的提示模板引导模型完成特定任务。

敏捷开发是一种以人为本、迭代增量的软件开发方法。2001年发布的敏捷宣言提出了四个核心价值观：个体和互动高于流程和工具、工作的软件高于详尽的文档、客户合作高于合同谈判、响应变化高于遵循计划。Scrum是最流行的敏捷框架，通过Sprint规划、每日站会、Sprint评审和回顾等仪式来管理迭代周期。Kanban方法则通过可视化工作流和限制在制品数量来实现持续交付。

联邦学习是一种保护数据隐私的分布式机器学习框架。传统机器学习需要将数据集中到服务器进行训练，而联邦学习让数据保留在本地设备上，各参与方在本地训练模型，仅将模型更新（梯度或参数）上传到中心服务器进行聚合。联邦平均（FedAvg）算法是最经典的聚合方法。差分隐私和安全多方计算常与联邦学习结合，进一步保护数据隐私。联邦学习在医疗、金融等隐私敏感领域具有重要应用价值。

模型压缩与加速技术对于将深度学习模型部署到资源受限的边缘设备至关重要。量化（Quantization）将浮点权重和激活值映射到低精度表示（如INT8、INT4），在保持模型精度的同时显著减少模型大小和推理延迟。剪枝（Pruning）移除网络中不重要的连接或神经元，稀疏化模型结构。知识蒸馏（Knowledge Distillation）让小模型（学生）模仿大模型（教师）的输出分布，实现模型压缩。神经架构搜索（NAS）自动寻找最优网络结构。

领域驱动设计（DDD）是一种以业务领域为核心的软件设计方法，由Eric Evans在其同名著作中系统阐述。限界上下文（Bounded Context）是DDD的核心概念，定义了特定领域模型的适用范围。聚合（Aggregate）是一组相关对象的集合，由聚合根保证业务规则的一致性。通用语言（Ubiquitous Language）是开发团队和业务专家共享的语言体系，贯穿代码、文档和对话，消除沟通歧义。

云安全是确保云计算环境中的数据、应用和基础设施免受威胁的综合策略和技术体系。共享责任模型是云安全的核心原则：云提供商负责云基础设施的安全，客户负责其在云中使用的数据和应用程序的安全。主要的云安全技术包括身份与访问管理（IAM）、加密（静态加密和传输加密）、安全组和网络ACL、Web应用防火墙（WAF）以及安全信息和事件管理（SIEM）。零信任架构强调从不信任、始终验证的安全理念。

人工智能（Artificial Intelligence，简称AI）是计算机科学的一个重要分支，旨在研究和开发能够模拟、延伸和扩展人类智能的理论、方法、技术及应用系统。其核心研究领域包括机器学习、自然语言处理、计算机视觉、语音识别和机器人技术等。近年来，随着深度学习技术的突破，特别是基于Transformer架构的大规模预训练模型的出现，人工智能在图像分类、机器翻译、语音合成等任务上取得了超越人类水平的表现。

云计算是一种通过互联网按需提供计算资源（包括服务器、存储、数据库、网络、软件等）的服务模式。用户无需自行购买和维护物理硬件，只需按使用量付费，从而大幅降低IT基础设施成本。云计算的服务模式主要包括基础设施即服务（IaaS）、平台即服务（PaaS）和软件即服务（SaaS）。Amazon Web Services（AWS）、Microsoft Azure和Google Cloud Platform是三大主流公有云服务提供商。

测试驱动开发（TDD）是一种编写代码前先编写测试的开发方法论。其核心循环是红-绿-重构：首先编写一个失败的测试（红），然后编写最简单的代码使测试通过（绿），最后在保持测试通过的前提下优化代码结构（重构）。单元测试验证代码的最小可测试单元（如函数或方法）的正确性。Mock和Stub等技术用于隔离被测试单元的外部依赖，确保测试的独立性和可重复性。

深度学习是机器学习的一个子集，利用多层神经网络从大规模数据中自动学习特征表示。卷积神经网络（CNN）擅长处理图像等网格结构数据，通过卷积层、池化层和全连接层的堆叠实现层次化的特征提取。循环神经网络（RNN）及其变体LSTM和GRU则适用于序列数据建模。Transformer架构通过自注意力机制彻底改变了自然语言处理领域，使得模型能够并行处理序列并捕获长距离依赖关系。

知识图谱是一种结构化的语义知识库，以图的形式描述现实世界中的实体及其关系。三元组（头实体-关系-尾实体）是知识图谱的基本组成单元。知识图谱在搜索引擎、推荐系统、智能问答和反欺诈等领域有广泛应用。知识图谱的构建包括信息抽取、实体对齐、关系推理和知识融合等步骤。知识图谱嵌入（KGE）将实体和关系映射到低维向量空间，保留语义相似性和结构信息。

迁移学习是将从一个任务或领域学到的知识应用到另一个相关任务或领域的方法。在深度学习中，迁移学习通常采用预训练加微调的策略：首先在大规模通用数据集上预训练模型，然后在目标任务的小规模数据集上进行参数微调。这种方法在数据标注成本高昂的场景下特别有效。域适应（Domain Adaptation）是迁移学习的一个重要分支，专注于源域和目标域分布不同的情况。
